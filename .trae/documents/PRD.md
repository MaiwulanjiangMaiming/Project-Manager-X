# 产品需求文档 (PRD) - Project Manager Pro

## 1. 项目概述

### 1.1 产品定位
一个现代化的 VS Code 扩展，用于直观管理开发项目。相比原版 vscode-project-manager，提供更美观、可拖拽、更直观的项目管理界面。

### 1.2 核心痛点
- 原版界面过于简陋，基于简单的 TreeView
- 不支持拖拽排序和分类管理
- 项目卡片信息展示不直观
- 缺乏现代化的视觉设计

## 2. 功能需求

### 2.1 核心功能
1. **项目收藏管理**
   - 保存当前文件夹/工作区为项目
   - 编辑项目名称、路径、标签
   - 删除项目

2. **标签分类系统**
   - 自定义标签（如 Work, Personal, Learning）
   - 项目可拥有多个标签
   - 按标签筛选项目

3. **自动检测项目**
   - 自动检测 Git/Mercurial/SVN 仓库
   - 自动检测 VS Code 工作区
   - 可配置检测路径和深度

4. **拖拽管理**
   - 拖拽项目到不同标签分类
   - 拖拽排序项目
   - 拖拽排序标签

5. **项目操作**
   - 在新窗口/当前窗口打开项目
   - 添加到工作区
   - 在文件管理器中显示
   - 重命名、删除

### 2.2 界面需求
1. **侧边栏 Webview 面板**
   - 现代化卡片式布局
   - 标签分组展示
   - 搜索过滤功能
   - 排序选项（名称、路径、最近使用）

2. **项目卡片**
   - 显示项目名称、路径、标签
   - Git 状态指示
   - 最近使用时间
   - 快捷操作按钮

3. **标签管理**
   - 颜色标识
   - 可折叠/展开
   - 拖拽排序

## 3. 设计方向

### 3.1 视觉风格
- **极简现代**: 干净的卡片设计，大量留白
- **深色主题优先**: 适配 VS Code 深色主题
- **微交互**: 悬停效果、拖拽反馈、平滑动画
- **色彩系统**: 使用 VS Code 主题色，保持原生感

### 3.2 布局结构
```
+----------------------------------+
|  [搜索框]              [设置 ⚙️]  |
+----------------------------------+
|  [全部] [标签1] [标签2] [+标签]  |
+----------------------------------+
|  📁 项目名称                      |
|  📂 /path/to/project    [🏷️标签] |
|  ─────────────────────────────── |
|  📁 另一个项目                    |
|  📂 /path/to/another   [🏷️工作]  |
+----------------------------------+
```

## 4. 技术架构

### 4.1 技术栈
- **前端**: React 18 + TypeScript
- **构建**: esbuild (极速构建，最小体积)
- **状态管理**: React Context + useReducer
- **拖拽**: @dnd-kit/core (轻量级，无依赖)
- **样式**: CSS Modules + CSS Variables

### 4.2 扩展架构
```
extension/
├── src/
│   ├── extension.ts          # 扩展入口
│   ├── webview/
│   │   ├── App.tsx           # React 应用根组件
│   │   ├── components/       # UI 组件
│   │   ├── hooks/            # 自定义 Hooks
│   │   └── styles/           # 样式文件
│   ├── core/
│   │   ├── projectManager.ts # 项目管理逻辑
│   │   ├── storage.ts        # 存储管理
│   │   └── autodetect.ts     # 自动检测
│   └── types/
│       └── index.ts          # 类型定义
├── package.json
└── tsconfig.json
```

## 5. 性能优化

### 5.1 体积优化
- 使用 esbuild 替代 webpack，构建产物更小
- Tree-shaking 移除未使用代码
- 按需加载组件
- 内联关键 CSS

### 5.2 运行优化
- 虚拟列表渲染大量项目
- 防抖搜索输入
- 缓存自动检测结果
- 增量更新 UI

## 6. 交互设计

### 6.1 拖拽交互
- 拖拽项目卡片时显示半透明预览
- 拖拽到标签区域时高亮目标标签
- 释放时平滑动画过渡

### 6.2 搜索交互
- 实时过滤，防抖 150ms
- 高亮匹配文本
- 支持按名称、路径、标签搜索

### 6.3 快捷操作
- 右键菜单：打开、在新窗口打开、编辑、删除
- 悬停显示快捷按钮
- 键盘快捷键支持
